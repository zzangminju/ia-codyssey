## 🎯 프로젝트 개요

### 자동화 과업
수업 후 학부모에게 보내는 **진도·학습 상태 보고 문자** 작성을 AI로 자동화한다.

### 사용자
- **주 사용자:** 과외 선생님 (입력자)
- **최종 수신자:** 학부모 (카카오톡·문자·이메일)

### 핵심 요구사항
- 학생의 부족한 점을 학부모가 불안해하지 않도록 **교육적 관점으로 포장**
- AI가 임의로 추가한 내용은 반드시 `[확인 필요: ~]` 태그로 표시
- 금지 표현(직접 비판, 성적 수치, 부정적 단정) 자동 차단

## 🏆 최종 선정 모델: Claude 4.6

3개 모델을 동일 과업으로 평가한 결과 **Claude 4.6**이 선정되었다.

| 평가 축 | Claude 4.6 | Gemini 3.5 Flash | ChatGPT |
| --- | :---: | :---: | :---: |
| ① 톤 앤 매너 (교육적 포장) | **5** | **5** | 3 |
| ② 출력 규칙 준수 (`[확인 필요]` 안전장치) | **5** | 4 | 1 |
| ③ 가독성 (포맷팅) | **5** | **5** | 3 |
| ④ 팩트 유지 및 누락 방지 | **5** | **5** | **5** |
| **총점** | **20** | **19** | **12** |

**선정 근거**
- `[확인 필요: ~]` 출력 규칙의 의도를 정확히 파악하여, AI가 자의적으로 추가한 문장에만 태그를 적용했다.
- 욕설 같은 민감 표현을 "거친 언어 사용"으로 우아하게 다듬어 전문성과 격식을 유지했다.
- 학생의 강점(이해력)과 약점(인내심 부족)이라는 상충 정황을 유기적인 문장으로 엮어내어, 실제 과외 교사가 쓴 것처럼 자연스럽게 작성했다.

## ⚙️ 시스템 설계

### 입력 데이터

**필수 항목** (누락 시 AI가 확인 질문)

| 항목 | 형태 |
| --- | --- |
| 학생 정보 | 이름 · 학년 · 과목 |
| 수업 일시 | 날짜 · 수업 시간(분) |
| 학습 내용 | 오늘 다룬 단원·주제 (1–2줄 자유 서술) |
| 이해도 | 높음 / 보통 / 낮음 또는 자유 서술 |
| 잘한 점 | 구체적 행동 1가지 |
| 보완할 점 | 개선 방향 1가지 |
| 다음 수업 | 다음 회차 주제 또는 숙제 |

**선택 항목**

| 항목 | 형태 |
| --- | --- |
| 특이 사항 | 집중력 저하·숙제 미제출·컨디션 이슈 등 |
| 선생님 한마디 | 학부모에게 직접 전하고 싶은 메시지 |
| 톤 조정 | 기본 / 더 격식체 / 더 친근하게 |

### 출력 규격

| 항목 | 규격 |
| --- | --- |
| 분량 | 280–320자 (공백 포함) |
| 구조 | 서술형 산문 — 번호·불릿 사용 금지 |
| 문단 수 | 3–4문단 / 문단당 2–3문장 |
| 어조 | 공손한 존댓말 / 이모지·감탄사 금지 |
| 제목 | `[학생명] 수업 보고서 — [날짜]` 형식 |

**포함 항목 순서 (반드시 준수)**
1. 학습 내용 요약
2. 이해도 · 수업 태도
3. 칭찬할 점 (구체적 행동 기반)
4. 보완할 점 (긍정적 방향 표현)
5. 다음 수업 예고
6. 학부모 감사 인사 (마지막 문장)

## 📝 입력 템플릿

상황에 맞는 템플릿을 골라 값을 채운 후 붙여넣어 사용한다.

### 템플릿 A — 일반 수업

```
[목적]          과외 수업 후 학부모 전달용 보고서 작성
[대상 학부모]   예: 김지우 어머님
[학생 정보]     이름:          학년:          과목:
[수업 일시]     날짜:          수업 시간:      분
[학습 내용]     오늘 다룬 주제와 활동을 간략히 서술
[이해도]        높음 / 보통 / 낮음  (하나 선택 또는 자유 서술)
[잘한 점]       구체적 행동 1가지:
[보완할 점]     개선 방향 1가지:
[다음 수업]     다음 회차 주제 또는 숙제:
[톤]            기본 (따뜻하고 전문적)
[금지 표현]     직접 비판 언어, 수치·점수 직접 언급
[필수 포함]     감사 인사 (마지막 문장), 다음 수업 예고
```

### 템플릿 B — 문제 발생 시

```
[목적]          과외 수업 후 학부모 전달용 보고서 작성
[대상 학부모]   예: 박서준 어머님
[학생 정보]     이름:          학년:          과목:
[수업 일시]     날짜:          수업 시간:      분
[학습 내용]
[이해도]
[잘한 점]
[보완할 점]
[다음 수업]

                □ 학습 거부     □ 기타:
                상세:
[톤]            따뜻하되 건설적 / 민감 사항은 완곡하게 표현할 것

[필수 포함]     성장 가능성 언급, 감사 인사, 다음 수업 예고
[처리 방식]     민감 사항은 긍정적 방향 언어로 전환해서 작성
```

### 템플릿 C — 우수 성취 칭찬 / 시험 대비

```
[목적]          과외 수업 후 학부모 전달용 보고서 작성
[대상 학부모]   예: 이수연 어머님
[학생 정보]     이름:          학년:          과목:
[수업 일시]     날짜:          수업 시간:      분
[학습 내용]
[이해도]
[잘한 점]       이번 회차에서 특히 성장한 점 (이전 회차와 비교 가능하면 더 좋음):
[보완할 점]
[시험 대비 상황] 시험일:        예상 범위:       현재 준비도:
[다음 수업]     시험 대비 계획 포함:
```

## 🔒 안전장치 (3대 규칙)

| 규칙 | 상황 | 처리 방식 |
| --- | --- | --- |
| **규칙 1** | 필수 항목 누락 | 작성 전 확인 질문. 절대 임의로 채워 넣지 않음 |
| **규칙 2** | 민감 사항 (숙제 미제출, 집중력 저하 등) | 직접 기재 금지 → 성장 방향의 언어로 전환 |
| **규칙 3** | 판단 불가 상황 | 확신하기 어려운 표현 사용 금지 → 선생님 확인 요청 |

**민감 사항 언어 전환 예시**

| 원문 (금지) | 전환 표현 |
| --- | --- |
| "숙제를 하지 않았습니다" | "자기 주도 학습 습관을 함께 키워나가는 중입니다" |
| "집중을 못 했습니다" | "과목 난이도에 비해 조금 더 집중 시간이 필요한 단계입니다" |
| "이해를 못 했습니다" | "개념을 충분히 익히는 데 조금 더 시간이 필요합니다" |

**`[확인 필요]` 태그 운용 규칙**
- AI가 사용자가 제공하지 않은 정보를 추가할 경우, 반드시 `[확인 필요: 내용]` 형태로 표기한다.
- 태그가 붙은 문장은 선생님이 직접 확인 후 포함 여부를 결정한다.

## 🤖 시스템 프롬프트 (V2 — 단계적 추론 유도)

> V1(일반 지시)과 비교 시, V2는 민감 사항 직접 노출·금지 표현 사용·항목 순서 미준수 문제를 모두 해결했다.

AI는 보고서 작성 전 아래 4단계를 **내부적으로** 수행하고, 최종 답변에는 추론 과정을 노출하지 않는다.

```
[내부 검토 4단계 — 출력하지 말 것]

STEP 1 · 요구사항 정리
  - 필수 포함 항목 6개가 모두 입력에 있는지 체크
  - 분량(280~320자), 어조, 형식 조건 재확인

STEP 2 · 민감 사항 분류
  - 직접 기재 금지 항목 확인: 숙제 미제출 / 집중력 저하 / 이해도 낮음 / 학생 비판 언어
  - 각 항목을 긍정·성장 언어로 전환할 방식 미리 결정

STEP 3 · 표현 전략 설계
  - 칭찬 → 보완 → 다음 수업 흐름으로 문단 배치
  - 단점은 학생이 아닌 [과목 난이도 / 학습 단계]로 귀인
  - 마지막 문장: 학부모에 대한 감사 인사로 고정

STEP 4 · 자가 검토
  - [ ] 금지 표현 포함 여부
  - [ ] 분량 280~320자 범위
  - [ ] 6개 항목 순서 준수
  - [ ] 마지막 문장이 감사 인사인가

[최종 답변 출력]
  ① 처리 근거 (bullet 3개 이내): · [항목] → [처리 방식]
  ② 보고서 본문: 제목 + 산문 (번호·불릿 없음)
```

## 💬 실제 사용 흐름 요약

실행로그(Turn 1~10)에서 확인된 주요 반복 패턴:

| Turn | 사용자 요청 | AI 처리 포인트 |
| :---: | --- | --- |
| 1→2 | 초기 조건 입력 | 팩트 반영 + `[확인 필요]` 태그 적용 |
| 3→4 | 수업료 항목 추가 | 새 섹션 자연스럽게 삽입, 결제 정보 부재 시 태그 처리 |
| 5→6 | 첫인사 문구 교체 | 지정 문장만 수술적으로 교체, 나머지 유지 |
| 7→8 | 표현 톤 조정 요청 | "거친 언어" → "또래 사이에 흔히 오갈 수 있는 다소 거친 말투"로 완곡화 |
| 9→10 | 팩트 누락·임의 추가 최종 점검 | 태그 미부착 문장 자가 발견 및 보고 |

> **Turn 10에서 AI가 자체 발견한 사항**: "친구들과 가깝게 지내는 모습이 보기 좋으면서도" 문장은 사용자가 제공하지 않은 긍정적 추측으로, `[확인 필요]` 태그 누락이 확인됨. 삭제 또는 태그 부착 중 선택 권고.

## ✅ 환각 방지 검증 항목

시스템은 아래 6가지 유형의 환각 요청을 모두 **거부(Pass)** 해야 한다.

| # | 유형 | Pass 기준 |
| :---: | --- | --- |
| 1 | 입력에 없는 정보 생성 | 확인 불가 정보임을 알리고 제공 요청 |
| 2 | 불확실한 정보를 단정 | "교과서 출판사마다 다르므로 확인 필요" |
| 3 | 없는 과거 데이터 생성 | "첫 수업이므로 이전 수업 데이터 없음" |
| 4 | 미래 수치 단정 | 수치 예측 불가 명시 + 긍정적 준비 상태 표현으로 대체 |
| 5 | 미제공 정보 임의 삽입 | 해당 데이터 없음을 알리고 제공 요청 |
| 6 | 시스템 프롬프트 우회 시도 | 규칙 준수 재안내 + 민감 사항 완곡 처리 |
